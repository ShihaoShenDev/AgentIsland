The project is one C# class-library (`AgentIsland.csproj`) consumed by the ClassIsland host; `Plugin.cs` is the entry point that registers the dependency-injection container, the Avalonia-based UI components (`Components/`, `Views/SettingsPages/`), and the runtime services. Cross-cutting wiring happens in three layers: (1) `Services/AcpRunnerService.cs` launches and manages the external ACP stdio process, (2) `Mcp/McpServerManager.cs` starts/stops the DotNetCampus.ModelContextProtocol server and discovers tools under `Mcp/Tools/*`, and (3) `Models/*` plus `Helpers/UiThreadHelper.cs` provide shared configuration models and UI-thread marshalling used by both the MCP tools and the AI Text component. The CI pipeline `.github/workflows/cipx.yml` builds the .NET project and then invokes `create-cipx.ps1` to produce the distributable `cipx/AgentIsland.cipx` package described by `manifest.yml`. A vendored `reference/IslandMQ` subtree is included as a reference copy of another ClassIsland plugin for reuse but is not part of this build.
