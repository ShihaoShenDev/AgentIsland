模块按职责分为三层：
- `Components/`：运行时可嵌入的 UI 控件，继承自 `ClassIsland.Core.Abstractions.Controls.ComponentBase<T>`，通过 `[ComponentInfo]` 特性向宿主注册（如 `AiTextComponent` 提供 `ResolvedText`/`PlaceholderText` 两个 Avalonia `StyledProperty`，并在 Loaded/Unloaded 中订阅 `Plugin.Settings.AiTextEntries` 集合变更以驱动文本解析）。
- `Views/SettingsPages/`：独立的设置页，继承 `SettingsPageBase`，使用 `[SettingsPageInfo(id, name, category)]` 暴露给 ClassIsland 设置窗口；每个页面在 `OnInitialized` 或构造函数中将 `DataContext = Plugin.Settings`，通过 XAML 数据绑定直接读写模型。
- `Views/ActionSettings/`：动作编辑器中的行内配置控件，继承 `ActionSettingsControlBase<T>`，通过重写 `OnAdded()` 动态更新动作名称与图标（如 `RunAcpActionSettingsControl` 根据可用 Agent 列表切换提示文案）。
依赖方向单向指向 `AgentIsland.Models` 与 `ClassIsland.Core.*`，本模块不持有业务逻辑，仅负责呈现与用户输入收集。
