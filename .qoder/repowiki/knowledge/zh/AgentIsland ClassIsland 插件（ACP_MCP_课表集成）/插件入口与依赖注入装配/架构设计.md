本模块是 AgentIsland 插件在 ClassIsland 宿主中的唯一入口与装配层，采用 Microsoft.Extensions.DependencyInjection + IHost 模式：
- `Plugin.cs` 通过 `[PluginEntrance]` 标记为插件入口，继承 `PluginBase`，在 `Initialize` 中完成配置持久化（`Settings.json`）、Sentry 遥测初始化、DI 容器注册（`AcpRunnerService`、`AgentIslandNotificationProvider`、4 个 SettingsPage、1 个 Action）以及订阅宿主 `AppStarted`/`AppStopping` 事件以启停 `McpServerManager`。
- `Services/AcpRunnerService.cs` 封装 ACP stdio JSON-RPC 协议：通过 `Process.Start` 拉起外部 Agent 进程，维护 `AcpAgentSession` 列表，实现 `initialize` → `session/prompt` 两阶段握手；`Dispose` 时优雅关闭 stdin 并在 5s 后强制 Kill。
- `Services/SentryTelemetryService.cs` 对 Sentry SDK 做可热插拔包装：监听 `AgentIslandSettings.PropertyChanged`，根据 `IsTelemetryActive` / `HasAgreedToPrivacyPolicy` / `CustomSentryDsn` 动态 Init/Shutdown，并提供 `WithInstrumentation`/`WithInstrumentationAsync` 装饰器统一埋点。
- `Automation/RunAcpAction.cs` 暴露 ClassIsland 自动化框架的 `RunAcpAction` 动作（`[ActionInfo("agentisland.acp.run", ...)]`），校验开关与 Agent 可用性后委托给 `AcpRunnerService.RunAgentAsync`，并通过 `AgentIslandNotificationProvider` 弹出通知。
依赖方向单向：Plugin → Services / Automation → Models/Mcp/Shared，Services 之间无相互引用，仅通过 DI 或 `IAppHost.GetService<SentryTelemetryService>()` 获取遥测实例。
