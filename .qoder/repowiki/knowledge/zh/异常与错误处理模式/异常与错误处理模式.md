本仓库采用 .NET 标准异常体系，结合 Microsoft.Extensions.Logging 和 Sentry 遥测进行错误捕获、上报与用户反馈。整体策略是：业务层抛出具体异常（如 InvalidOperationException、ArgumentException），MCP 工具层用 try/catch 包裹并返回结构化结果或记录日志，插件生命周期在顶层 catch 后通过 ILogger 和 SentryTelemetryService 上报。

1. 使用的系统与框架
- 异常类型：使用 .NET 内置异常（InvalidOperationException、ArgumentException）表达参数/状态错误；未定义自定义 Exception 派生类。
- 日志：Microsoft.Extensions.Logging，通过 IAppHost.GetService<ILogger<T>> 获取实例，统一 LogInformation/LogWarning/LogError/LogDebug。
- 遥测：SentryTelemetryService 封装 Sentry SDK，提供 CaptureException、AddBreadcrumb、WithInstrumentation/WithInstrumentationAsync 等 API，按用户隐私设置动态启停。
- 无全局异常中间件或 panic/recover 机制；错误传播以 throw/catch + 结构化返回值为主。

2. 关键文件与位置
- Plugin.cs：插件入口，在 AppStarted/AppStopping 中启动/停止 MCP 服务，顶层 try/catch 记录错误并上报。
- Services/SentryTelemetryService.cs：Sentry 初始化、配置、异常捕获与事务包装的统一入口。
- Mcp/Tools/ScheduleTools.cs：课表相关工具，对可变操作（SwapClasses）try/catch 返回 SwapResult(false, ex.Message)，对只读操作通过 WithInstrumentation 自动埋点。
- Automation/RunAcpAction.cs：自动化动作入口，对前置条件校验失败直接 throw InvalidOperationException，由上层捕获。

3. 架构与约定
- 参数校验：在工具方法入口处抛 ArgumentException（如日期格式、缺失参数），调用方不吞掉该异常，交由上层统一处理。
- 业务状态错误：在 ActionBase 子类中对“功能未启用”“Agent 不存在/已停用”等场景抛 InvalidOperationException，配合 LogWarning 记录上下文。
- 可恢复错误：MCP 工具对可能失败的副作用操作（如交换课程）用 try/catch 捕获并返回 Result 对象（SwapResult、ScheduleResult 等），避免异常冒泡到 MCP 协议层。
- 遥测集成：所有对外暴露的 MCP 工具优先通过 telemetry.WithInstrumentation(name, action) 包装，自动添加面包屑、Transaction 并在异常时上报 InternalError。
- 顶层兜底：Plugin 的 OnAppStarted/OnAppStopping 以及 McpServerManager 的 StartAsync/StopAsync 均 catch Exception，同时写入 ILogger 和 Sentry，再向上抛出或忽略，保证宿主进程稳定。

4. 开发者应遵循的规则
- 不要吞掉 ArgumentException/InvalidOperationException：这些表示调用方错误或不可恢复状态，应让上层决定如何呈现给用户。
- 对可能失败的外部调用（ClassIsland API、文件系统、网络）使用 try/catch 返回 Result 类型，而不是向上抛异常。
- 需要埋点的公开方法统一通过 SentryTelemetryService.WithInstrumentation* 包装，确保异常被捕获且附带 context/tool 标签。
- 所有异常路径都应通过 ILogger 记录至少一条 Warning/Error 日志，包含必要上下文（参数值、目标名称等）。
- 不要在 UI 线程外访问 ClassIsland 服务：如需跨线程，使用 UiThreadHelper.RunOnUi 包裹，并在外层 try/catch 捕获异常。
- 新增工具方法时，先判断参数有效性并抛 ArgumentException，再执行业务逻辑；对副作用操作返回 Result，对纯查询返回数据模型。
