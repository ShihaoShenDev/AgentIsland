本仓库采用 **双包管理器** 的依赖管理模式：.NET 项目通过 NuGet 管理运行时与 SDK 依赖，Node.js 子模块（`.mimocode`）通过 npm 管理。两者均使用锁定文件保证可复现构建。

### 1. .NET 依赖（NuGet）
- 核心清单位于 `AgentIsland/AgentIsland.csproj`，目标框架为 `net8.0-windows`。
- 关键包及版本：
  - `ClassIsland.PluginSdk`：通过属性 `ClassIslandPluginSdkVersion=1.7.106.2-dev-v2` 集中声明，避免散落的版本号。
  - `DotNetCampus.ModelContextProtocol`、`AgentClientProtocol`、`Sentry`：以 `<PackageReference>` 形式直接声明。
- **手动复制运行时 DLL**：`System.Text.Json`、`System.Text.Encodings.Web`、`System.IO.Pipelines`、`Sentry.dll` 等通过 `<None Include="$(NuGetPackageRoot)..." CopyToOutputDirectory="PreserveNewest">` 显式复制到输出目录，说明宿主 ClassIsland 对运行时程序集有特定加载要求。
- 无全局 `nuget.config` / `NuGet.Config`，未配置私有源或代理；所有包均来自公共 NuGet 源。
- 无 `packages/` 文件夹，依赖由 NuGet 缓存管理，符合现代 SDK-style 项目约定。

### 2. Node.js 依赖（npm）
- 仅 `.mimocode/package.json` 声明一个生产依赖：`@mimo-ai/plugin@0.1.3`。
- 配套 `package-lock.json`（lockfileVersion: 3）锁定完整依赖树，包含原生平台可选包（`msgpackr-extract-*`）。
- 无 `node_modules` 提交到版本库（被 `.gitignore` 忽略），但存在本地 `plans/` 目录，表明该子模块可能用于 AI 辅助生成脚本而非运行时依赖。

### 3. 构建与发布中的依赖处理
- `build.ps1` / `build-release.ps1` 调用 `dotnet build -c Debug|Release`，不执行额外还原步骤，依赖还原由 MSBuild/NuGet 自动完成。
- CI 配置（`.github/workflows/cipx.yml`）未在已读范围内发现自定义 NuGet/npm 源设置，推测沿用默认行为。
- 插件产物 `cipx/AgentIsland.cipx` 由打包流程生成，其中应包含上述手动复制的运行时 DLL。

### 4. 开发者应遵循的规则
- 新增 .NET 依赖时，优先在 `AgentIsland.csproj` 中用 `<PackageReference>` 声明；若宿主需要特定运行时 DLL 随插件分发，需同步添加对应的 `<None Include="$(NuGetPackageRoot)...">` 条目。
- 统一通过 `ClassIslandPluginSdkVersion` 属性管理 ClassIsland SDK 版本，避免硬编码散落各处。
- 如需引入新的 npm 依赖，应在 `.mimocode/package.json` 中添加并更新 `package-lock.json`，不要手动编辑 lock 文件。
- 不使用 vendoring（无 vendor/、third_party/ 目录），也不配置私有 NuGet/npm 源；所有第三方包来自公共注册表。
