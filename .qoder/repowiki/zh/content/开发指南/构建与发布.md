### 构建脚本
- **build.ps1（调试构建）**
  - 作用：终止正在运行的 ClassIsland 桌面进程，编译调试版本并启动 ClassIsland 进行联调。
  - 输出路径：$PWD\AgentIsland\bin\Debug\net8.0-windows
  - 适用场景：开发阶段快速迭代与调试
- **build-release.ps1（发布构建）**
  - 作用：编译发布版本并启动 ClassIsland 进行联调。
  - 输出路径：$PWD\AgentIsland\bin\Release\net8.0-windows
  - 适用场景：发布前功能验证
- **cipx.ps1（插件打包）**
  - 作用：创建 ClassIsland 插件包（.cipx），内部调用 dotnet publish -c Release -p:CreateCipx=true。
  - 输出路径：$PWD\AgentIsland\cipx
  - 适用场景：生成可分发的插件包
- **cipx-debug.ps1（调试打包）**
  - 作用：使用 Debug 模式创建插件包，便于开发测试。
  - 输出路径：$PWD\AgentIsland\cipx

### 清单文件 manifest.yml
- 字段说明
  - id：插件唯一标识符，建议采用反向域名风格（ShihaoShen.AgentIsland）。
  - name：插件显示名称（AgentIsland）。
  - description：插件描述（你的课表，智能进化）。
  - entranceAssembly：入口程序集文件名（AgentIsland.dll）。
  - url：插件主页或仓库地址（https://github.com/ShihaoShenDev/AgentIsland）。
  - version：插件版本号，遵循语义化版本规范（0.1.0.0）。
  - apiVersion：兼容的 ClassIsland API 版本（2.0.0.0）。
  - author：作者信息（Shihao Shen）。
  - repoOwner：仓库所有者（ShihaoShenDev）。
  - repoName：仓库名称（AgentIsland）。
  - assetsRoot：资源根目录名（main）。
  - supportedOSPlatforms：受支持的操作系统平台列表（Windows）。
  - dependencies：可选依赖声明（如 lrs2187.sai）。
- 资源文件管理
  - 清单中的 assetsRoot 指定了资源根目录，构建系统会据此将资源文件纳入插件包。
  - 项目文件中通过 None 项将清单、说明文档与图标复制到输出目录，确保打包时可被找到。
- 版本与平台
  - version 与 apiVersion 需与 ClassIsland 运行时保持兼容。
  - supportedOSPlatforms 限制插件可在哪些平台上运行。

### 构建与发布模式
- Debug 模式
  - 使用 build.ps1 进行快速迭代与联调。
  - 自动终止 ClassIsland 进程后执行 dotnet build，并以调试输出目录（$PWD\AgentIsland\bin\Debug\net8.0-windows）作为扩展加载路径启动 ClassIsland。
  - 适用于开发阶段，不建议用于打包发布。
- Release 模式
  - 使用 build-release.ps1 进行发布级构建。
  - 执行 dotnet build -c Release，并将发布输出目录（$PWD\AgentIsland\bin\Release\net8.0-windows）作为扩展加载路径启动 ClassIsland。
  - 适合验证打包前的功能完整性。
- 打包模式
  - 使用 cipx.ps1 生成 cipx 包，内部通过 dotnet publish -c Release -p:CreateCipx=true 触发打包目标。
  - 打包产物位于 AgentIsland/cipx 目录，可直接用于 ClassIsland 安装或提交至插件市场。
  - 使用 cipx-debug.ps1 可进行 Debug 模式的打包测试。

## 性能考虑
- 构建优化
  - 使用 Release 模式构建以获得更优的运行时性能与体积。
  - 合理设置 CopyToOutputDirectory 策略，避免不必要的文件复制。
- 运行时优化
  - 按需引入依赖，减少运行时负载。
  - 对日志与遥测进行开关控制，避免在生产环境产生过多开销。

## 故障排查指南
- 构建失败
  - 确认已安装 .NET 8 SDK，且环境变量 PATH 中包含 dotnet。
  - 检查 ClassIsland 开发环境是否按官方文档正确配置。
  - 确认 $env:ClassIsland_DebugBinaryDirectory 环境变量指向正确的 ClassIsland 调试二进制文件位置。
- 打包失败
  - 确认 AgentIsland/manifest.yml 存在且格式正确。
  - 确认入口程序集名称与清单中 entranceAssembly 一致。
  - 检查 assetsRoot 指向的资源目录是否存在。
  - 确认输出路径权限正常。
- 运行失败
  - 确保 ClassIsland 进程未被占用（脚本会自动终止）。
  - 检查端口占用与防火墙设置。
  - 查看 ClassIsland 日志与 Sentry 遥测信息定位问题。
  - 确认插件输出路径正确（AgentIsland/bin/*/net8.0-windows）。
